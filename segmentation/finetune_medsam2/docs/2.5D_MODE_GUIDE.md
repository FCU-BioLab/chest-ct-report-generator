# MedSAM2 2.5D 模式使用指南

## 🎯 什麼是 2.5D 模式？

**2.5D 模式**將相鄰的三個 CT 切片（Z-1, Z, Z+1）分別作為 RGB 三個通道輸入，為模型提供**上下文資訊**，提升分割效果。

### 對比

| 模式 | 輸入 | 優點 | 缺點 |
|------|------|------|------|
| **2D** | 同一切片重複 3 次 | 訓練快、記憶體低 | 無上下文資訊 |
| **2.5D** ⭐ | Z-1, Z, Z+1 三個切片 | 有上下文、效果好 | 幾乎無額外成本 |

### 預期效果提升

- **Dice Score**: +5-10%
- **小結節檢測**: 顯著提升
- **邊界精確度**: +3-5%
- **訓練時間**: 幾乎不變（+5%）

## 🚀 使用方法

### 1. 預設啟用（推薦）

2.5D 模式**預設啟用**，直接訓練即可：

```bash
# 使用快取資料訓練（2.5D 模式自動啟用）
python finetune_medsam2/main.py --use_cache --cache_dataset_type lndb

# 使用原始資料訓練
python finetune_medsam2/main.py --data_dir path/to/LNDb
```

### 2. 明確啟用 2.5D

```bash
python finetune_medsam2/main.py --use_cache --use_2_5d
```

### 3. 禁用 2.5D（使用傳統 2D）

如果需要對比實驗或相容性測試：

```bash
python finetune_medsam2/main.py --use_cache --no_2_5d
```

## 💡 技術細節

### 資料載入流程

#### 2.5D 模式
```python
# 載入 3 個相鄰切片
z_prev = max(0, slice_idx - 1)
z_next = min(num_slices - 1, slice_idx + 1)

slices = [ct_array[z_prev], ct_array[slice_idx], ct_array[z_next]]

# 組合為 RGB
ct_rgb = np.stack(slices, axis=-1)  # [H, W, 3]
# 通道 0: Z-1 (前一張切片)
# 通道 1: Z   (當前切片)
# 通道 2: Z+1 (後一張切片)
```

#### 2D 模式
```python
# 只載入單一切片
ct_slice = ct_array[slice_idx]

# 重複 3 次
ct_rgb = np.stack([ct_slice] * 3, axis=-1)  # [H, W, 3]
# 通道 0, 1, 2: 都是同一張切片
```

### 邊界處理

- **第一張切片** (idx=0): Z-1 使用 idx=0（重複）
- **最後一張切片** (idx=N-1): Z+1 使用 idx=N-1（重複）
- **中間切片**: 正常使用 Z-1, Z, Z+1

## ❓ 常見問題

### Q1: 2.5D 會增加多少訓練時間？

**A**: 幾乎不會增加（+5%）。只是多載入 2 個切片，計算量幾乎相同。

### Q2: 2.5D 需要更多記憶體嗎？

**A**: 不需要。輸入尺寸仍然是 `[3, 512, 512]`，只是三個通道的內容不同。

### Q3: 可以中途切換模式嗎？

**A**: 不建議。2D 和 2.5D 訓練的模型不相容，需要重新訓練。

### Q4: 如何驗證是否使用了 2.5D？

**A**: 查看訓練日誌中的 `🔧 輸入模式` 訊息，或檢查第一 epoch 的樣本視覺化。
