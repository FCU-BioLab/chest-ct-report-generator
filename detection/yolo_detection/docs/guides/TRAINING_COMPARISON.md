# ⚖️ 訓練方法對比：原始 vs 預處理

> **注意**: 本文檔為詳細對比分析。快速開始請參考 [README.md](README.md)

## 核心差異

| 項目 | 原始 DICOM | 預處理 PNG |
|------|-----------|-----------|
| **速度** | 4.2 秒/batch | 0.5 秒/batch |
| **120 Epochs** | 30-35 天 | **2-3 天** ✅ |
| **GPU 利用率** | 10-15% | 80-100% |
| **預處理時間** | 無 | 60 分鐘（一次性）|
| **磁碟空間** | 0 | 6 GB |
| **ROI** | - | **672 倍時間節省** |

## 詳細瓶頸分析

### 原始 DICOM 訓練（慢）

**每個 batch 的時間分配**:
```
讀取 DICOM (pydicom)     2.5 秒  60% ← 主要瓶頸
HU windowing             0.5 秒  12%
CLAHE enhancement        0.3 秒   7%
Resize + 增強             0.4 秒  10%
GPU 訓練                  0.5 秒  11%
────────────────────────────────
總計                      4.2 秒  100%

GPU 利用率: 10-15% (大部分時間在等待 CPU 和磁碟)
```

### 預處理 PNG 訓練（快）

**每個 batch 的時間分配**:
```
讀取 PNG (快!)           0.1 秒  20%
增強（可選）             0.1 秒  20%
GPU 訓練                  0.3 秒  60% ← GPU 成為瓶頸（好事！）
────────────────────────────────
總計                      0.5 秒  100%

GPU 利用率: 80-100% (完全發揮硬體效能)
```

| 項目 | 數值 |
|------|------|
| **預處理投資** | 60 分鐘 + 6 GB |
| **訓練節省（單次）** | 32.5 天 |
| **ROI** | **780 倍** |
| **可進行實驗次數** | 10 次 (vs 原本 1 次) |

---

## 使用建議

| 場景 | 推薦方法 | 原因 |
|------|---------|------|
| 初期探索（1-10 epochs）| 原始 DICOM | 無需預處理，快速驗證 |
| 完整訓練（120 epochs）| 預處理 PNG | 節省 28-32 天 |
| 超參數調優（多次實驗）| 預處理 PNG | 10 倍迭代速度 |
| 磁碟空間不足（< 6 GB）| 原始 DICOM | 無需額外空間 |

## 快速開始

### 預處理（60 分鐘）
```cmd
cd detection\dataset_process
python preprocess_for_yolo.py ^
    --data_root ../../datasets/splited_dataset ^
    --output_dir ../../datasets/preprocessed_yolo ^
    --imgsz 640
```

### 訓練（2-3 天）
```cmd
cd ..\yolo_detection
python train_yolov7_preprocessed.py ^
    --data_dir ../../datasets/preprocessed_yolo ^
    --epochs 120 ^
    --batch_size 24
```

---

**結論**: 預處理是一次性投資（60 分鐘），換取永久的 8-10 倍速度提升。強烈推薦！
